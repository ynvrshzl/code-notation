## 1.0 Fundamental series
The fundamental computational model, which is the fundmaental model of all computer programs, is described as a transformation machine:

```math
\sum o = x(i)
```
- where output "o" is the result
- of input "i"
- transformed by the unknown "x"

This tangibly describes the fundamental model of a computer program. In this sense, the input is the "idea" or problem to solve, in which the computer program exists solely as an executable product, not the architectural design.

The output could be the computer program itself, or the output of a transformation.

## 2.0 Fundamental inversion feedback
If $o = x(i)$ is true for computational models. Then the reverse is true for the human operator, and engineer. Arguably, a more fundamental feedback model, which harmonizes with the fundamental series transformation machine.

```math
\sum i = o(x)
```

Where $i$ is the human input source (or a recursive input from another transformation $x$)...

### 1.1 Abstraction layer
```math
a^n = x(i)^n
```
In programming, an abstraction layer is a structural foundation of computational systems. In this case, an abstraction layer is the fractal recursion of the fundamental model $o = x(i)$ where $x$ is unfolded, and developed into the first stage of transformation. 

In classical programming, this is simply the act of turning a block of code, into a function. We not only increase system readability, but introduce verbosity: (input, output, function body). This creates a logarithmic increase of system scale...[^1]
